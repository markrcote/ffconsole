## Component Inventory

Components introduced or modified in this phase:

### 1. "Start Battle" Button (new element in existing section)

- **Element:** `<button id="start-battle-btn" class="mechanic-btn mechanic-btn--primary">Start Battle</button>`
- **Placement:** Inside `.tests-section > .test-buttons` div, alongside `#test-luck`
- **Size:** `min-height: 48px`, `flex: 1`, `min-width: 90px` — inherits `.mechanic-btn` rules
- **Visual state — resting:** `background: var(--ink-color)` / `color: var(--paper-bg)` (primary)
- **Visual state — hover:** `background: var(--accent-red)`, `border-color: var(--accent-red)`
- **Visual state — active:** `transform: scale(0.95)` — inherits `.stat-btn:active` not mechanic-btn; note: mechanic-btn has no `:active` scale rule currently — use `transform: translateY(0)` consistent with `.action-btn:active`
- **Touch:** `touch-action: manipulation` — already declared for `.mechanic-btn` in existing CSS

Source: CONTEXT.md D-03; `css/style.css` `.mechanic-btn`, `.mechanic-btn--primary`.

### 2. Tests Section Renamed to "Actions"

- **Before:** `<section class="mechanics-section tests-section">` with `<h2 class="mechanics-title">Tests</h2>`
- **After:** `<h2 class="mechanics-title">Actions</h2>` — heading text only changes; class `tests-section` may be renamed to `actions-section` for consistency (Claude's discretion per CONTEXT.md)
- **Recommended:** Rename class to `actions-section` for semantic clarity; no CSS rule currently targets `.tests-section` by name (styles come from `.mechanics-section`)

Source: CONTEXT.md D-01, specifics block.

### 3. Combat Section Removed from index.html

- **Before:** `<section class="mechanics-section combat-section">` with all inline combat HTML (lines 76–130 of `index.html`)
- **After:** Section entirely removed. All global IDs that lived here (`combat-active`, `combat-setup`, `enemy-name`, `enemy-skill`, `enemy-stamina`, `start-combat`, `combat-status`, `player-stamina-fill`, `enemy-stamina-fill`, `player-stamina-value`, `enemy-stamina-value`, `enemy-stamina-label`, `combat-round-result`, `combat-result`, `roll-round`, `flee-combat`, `combat-summary`) are removed from `index.html` and will live only in modal-generated DOM.
- **Battle History section remains:** `<section class="mechanics-section combat-history-section" id="combat-history-section">` stays in `index.html` unchanged.

Source: CONTEXT.md D-02, D-04; `index.html` lines 76–135.

### 4. battleModal.js — Modal Overlay Skeleton

- **Pattern:** Follows `charCreate.js` create-on-open / destroy-on-close — creates `div.modal-overlay.active`, appends to `document.body`, removes via `overlay.remove()` on close.
- **Exported function:** `openBattleModal(getState, callbacks)` — called from `app.js`
- **Optional stub:** `closeBattleModal()` — Phase 7 will fully implement; skeleton may export a no-op stub or leave teardown to Phase 7 (Claude's discretion per CONTEXT.md D-06/D-08)

**Modal inner HTML structure (full structure per D-07):**

```
div.modal-overlay.active
  └── div.modal.battle-modal
        ├── h2.modal-title  "Combat"
        ├── div.combat-setup (visible)
        │     ├── combat-input-row: enemy name
        │     ├── combat-input-row: enemy skill
        │     ├── combat-input-row: enemy stamina
        │     └── button.mechanic-btn.mechanic-btn--primary "Start Combat"
        ├── div.combat-active (hidden)
        │     ├── p.combat-status
        │     ├── div.stamina-bar-group
        │     ├── div.combat-round-result
        │     ├── p.mechanic-result[aria-live="polite"]
        │     └── div.combat-actions (Roll Round + Flee buttons)
        └── div.combat-summary (hidden)
```

- `combat-active` and `combat-summary` sections start with `hidden` attribute — Phase 7 wires the show/hide transitions.
- All element references use `container.querySelector()` — no `document.getElementById()` calls.

Source: CONTEXT.md D-06, D-07; `index.html` existing combat-section structure; `css/style.css` existing `.modal-overlay`, `.modal`, `.combat-setup`, `.combat-active`, `.combat-summary` rules.

### 5. battle.js — Container-Scoped Refactor

- **All** `document.getElementById()` calls replaced with `container.querySelector()` where `container` is the argument passed in.
- `updateStaminaBars()` and other helpers that query global IDs must accept container or pre-queried element refs as arguments. Recommended approach (Claude's discretion): pass the container element and let each helper call `container.querySelector()` internally — avoids threading multiple element refs through call signatures.
- No new visual changes — this is an internal refactor with no user-visible differences.

Source: CONTEXT.md D-04, D-05.
