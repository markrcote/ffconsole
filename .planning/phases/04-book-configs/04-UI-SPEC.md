- Element: `<section class="mechanics-section book-mechanics-section" id="book-mechanics-section">`

- Buttons: 40x40px, `border: 2px solid var(--ink-color)`, `border-radius: 4px`
- Touch target: `touch-action: manipulation` (already on `.stat-btn` in CSS)

### 3. Resource Row (same as extra stat row, visual treatment identical)

- Provisions and Fuel for Book 30
- Same `.stat-row` pattern
- No upper-cap visual indicator needed (resources have no fixed maximum in FF)

### 4. Checklist Group

- Container: `<div class="checklist-group">`
- Group label: `<p class="mechanics-title checklist-group__label">` — MedievalSharp 0.85rem, uppercase, letter-spacing 2px, `color: var(--ink-light)`
- Item list: `<ul class="checklist-items">` — `list-style: none`, `padding: 0`
- Item row: `<li class="checklist-item">` — `display: flex`, `align-items: center`, `gap: 8px`, `padding: 6px 0`, `border-bottom: 1px dashed var(--paper-shadow)`

- Item label: `<label>` — Caveat 1.1rem, `color: var(--ink-color)`, `cursor: pointer`
- Checked item: label gets `text-decoration: line-through`, `color: var(--ink-light)`
- Last item: `border-bottom: none`
- Margin between groups: 16px top

### 5. Superpower Picker (character creation step — `charCreate.js`)

- Renders as a new step in the existing character creation modal
- Step label: `.char-create-section-label` — "Choose Your Superpower" — MedievalSharp 0.875rem, uppercase
- Option list: `<div class="superpower-options">` — `display: flex`, `flex-direction: column`, `gap: 8px`

- Confirmation: the existing modal "Confirm" button advances only when a superpower is selected; button is disabled (`opacity: 0.3`, `cursor: not-allowed`) until selection made
- No deselection needed — once picked, selection is final within the step

### 7. Book Mechanics Section Subtitle

## Interaction Contracts

### Extra Stat / Resource +/- Buttons

- Tap `-`: decrease value by 1; clamp at `min` (0 for all defined stats/resources)
- Tap `+`: increase value by 1; clamp at `max` if defined (Hero Points: no max per book rules; Tabasha: max 4; Provisions/Fuel: no max)
- Long-press behavior: NOT applicable to book mechanic stats — no bonus-above-initial concept. Standard tap only.
- Disabled appearance: `opacity: 0.3`, `cursor: not-allowed` when at boundary
- State saves immediately on each change: calls `onMechanicsChange()` callback

### Checklist Items

- Tap checkbox or label: toggles checked state
- Checked visual: label gains `text-decoration: line-through` + `color: var(--ink-light)`
- Unchecked visual: label returns to `color: var(--ink-color)`, no strikethrough
- State saves immediately on toggle: calls `onMechanicsChange()` callback

### Superpower Selection (char create)

- Tap option button: deselects any previous selection, selects tapped option (adds `mechanic-btn--primary`)
- Confirm button: enabled only when a superpower is selected
- Confirm tap: passes `{ superpower: selectedValue }` to `onComplete` alongside stats + name
- Back/cancel: discards superpower selection, returns to name step (standard modal cancel behavior)

### Section Show/Hide

- When `getBookConfig(bookNumber)` returns a config with non-empty `extraStats`, `resources`, `checklists`, or `superpower`: remove `hidden` from `#book-mechanics-section`
- When no supported book is active (base sheet): add `hidden` to `#book-mechanics-section`
- No animation on show/hide — instant toggle to match existing `.hidden` behavior
