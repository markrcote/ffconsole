Create `js/ui/bookMechanics.js` as a new ES module. Follows D-17 pattern: does NOT import app.js. Receives all state via function parameters.

**Exported function signature:**
```javascript
export function renderBookMechanics(container, bookConfig, mechanicsState, onMechanicsChange)
```

Parameters:
- `container` â€” the `#book-mechanics-section` DOM element
- `bookConfig` â€” the config object from getBookConfig() (has extraStats, resources, checklists, superpower keys)
- `mechanicsState` â€” object like `{ stat_hero_points: 3, checklist_clues_clue-1: true, superpower: 'Flying' }` â€” current persisted mechanics values
- `onMechanicsChange(updatedMechanicsState)` â€” callback invoked on every change with the full updated mechanics dict

**Implementation details:**

1. **Guard clause:** If `bookConfig` has no extraStats, resources, checklists, and no superpower key, set `container.hidden = true` and return early. Otherwise, remove the `hidden` attribute.

2. **Clear container** and rebuild inner HTML. Start with a subtitle:
   ```html
   <h3 class="mechanics-title">{bookConfig.subtitle || bookConfig.bookNumber === 17 ? 'F.E.A.R. Mechanics' : bookConfig.bookNumber === 30 ? 'Chasms of Malice' : 'Book Mechanics'}</h3>
   ```

3. **extraStats rendering** â€” For each stat in `bookConfig.extraStats`, render a `.stat-row` with the same DOM structure as the base stats (label + stat-controls with -/+ buttons + value display). Key differences from base stats:
   - No initial/current split â€” single value displayed (no `/` separator)
   - Value reads from `mechanicsState['stat_' + stat.id]` or falls back to `stat.initial`
   - `-` button: decrease by 1, clamp at `stat.min` (usually 0). Disabled at min boundary (`opacity: 0.3; cursor: not-allowed` via `.stat-btn:disabled`).
   - `+` button: increase by 1, clamp at `stat.max` if defined. If `stat.max === null`, no upper limit. Disabled at max boundary.
   - No long-press / bonus behavior â€” standard click only
   - On change: update `mechanicsState['stat_' + stat.id]`, re-render value + button disabled states, call `onMechanicsChange(mechanicsState)`
   - Use unique IDs for elements: `bm-stat-${stat.id}-current`, `bm-stat-${stat.id}-decrease`, `bm-stat-${stat.id}-increase`

4. **resources rendering** â€” Identical to extraStats rendering. For each resource in `bookConfig.resources`:
   - Value reads from `mechanicsState['resource_' + resource.id]` or falls back to `resource.initial`
   - Same +/- button pattern, same boundary clamping
   - Use IDs: `bm-resource-${resource.id}-current`, `bm-resource-${resource.id}-decrease`, `bm-resource-${resource.id}-increase`

5. **checklists rendering** â€” For each checklist in `bookConfig.checklists`:
   - Render a `.checklist-group` div with:
     - `<p class="mechanics-title checklist-group__label">{checklist.label}</p>`
     - `<ul class="checklist-items">` containing one `<li class="checklist-item">` per item
   - Each item: `<input type="checkbox" id="bm-check-{checklistId}-{itemId}">` + `<label for="bm-check-{checklistId}-{itemId}">{item.label}</label>`
   - Checked state reads from `mechanicsState['checklist_' + checklist.id + '_' + item.id]` (boolean, default false)
   - If checked, add `checked` class to the `.checklist-item` li element
   - On checkbox change: update `mechanicsState['checklist_' + checklist.id + '_' + item.id]` to the checkbox checked state, toggle `.checked` class on the parent `li`, call `onMechanicsChange(mechanicsState)`

6. **superpower display** â€” If `bookConfig.superpower` exists AND `mechanicsState.superpower` is set (truthy string):
   - Render read-only display:
     ```html
     <div class="superpower-display">
       <p class="mechanics-title">Superpower</p>
       <p class="superpower-display__value">{mechanicsState.superpower}</p>
     </div>
     ```
   - No edit affordance (D-15: read-only on the sheet)

7. **Event binding** â€” After setting innerHTML, query all buttons and checkboxes and attach event listeners. Use event delegation on the container where practical:
   - For +/- buttons: bind click handlers that modify mechanicsState and call onMechanicsChange
   - For checkboxes: bind change handlers that update mechanicsState and call onMechanicsChange
   - All buttons get `touch-action: manipulation` (already inherited from CSS global selector `.stat-btn`)

8. **Helper: internal re-render function** â€” Create an internal `updateDisplay()` function that re-renders just the values and disabled states (not the full DOM), so button clicks feel snappy. Full DOM rebuild only happens on initial call.

   ```html
   <div id="cc-superpower-step" style="display:none;">

   </div>
   ```

3. **Add local state** â€” Add `let selectedSuperpower = null;` alongside the existing `let selectedBook = null;` and `let rolledStats = null;`.

4. **On book selection** â€” In the existing `bookList.addEventListener('click', ...)` handler, after the book is selected, check if the selected book has a superpower config. Add an async call:
   ```javascript
   // Check for superpower config
   const superpowerStep = overlay.querySelector('#cc-superpower-step');
   const superpowerOptions = overlay.querySelector('#cc-superpower-options');
   if (superpowerStep && superpowerOptions) {
       const bookCfg = await getBookConfig(num);
       if (bookCfg && bookCfg.superpower && bookCfg.superpower.options) {
           selectedSuperpower = null; // Reset on book change
           superpowerStep.style.display = '';
           superpowerOptions.innerHTML = bookCfg.superpower.options.map(opt =>
               `<button class="superpower-option mechanic-btn" data-power="${opt}">${opt}</button>`
           ).join('');
           // Disable confirm until superpower selected
           if (rolledStats) confirmBtn.disabled = true;
       } else {
           selectedSuperpower = null;
           superpowerStep.style.display = 'none';
           superpowerOptions.innerHTML = '';
           // Re-enable confirm if stats rolled
           if (rolledStats) confirmBtn.disabled = false;
       }
   }
   ```
   Note: The book list click handler needs to become async. Wrap the existing handler body or use an async IIFE inside.

5. **Superpower option click handler** â€” Use event delegation on `#cc-superpower-options`:
   ```javascript
   const superpowerOptionsEl = overlay.querySelector('#cc-superpower-options');
   if (superpowerOptionsEl) {
       superpowerOptionsEl.addEventListener('click', (e) => {
           const btn = e.target.closest('.superpower-option');
           if (!btn) return;
           selectedSuperpower = btn.dataset.power;
           // Update visual selection
           superpowerOptionsEl.querySelectorAll('.superpower-option').forEach(b => {
               b.classList.toggle('mechanic-btn--primary', b === btn);
           });
           // Enable confirm if stats are rolled
           if (rolledStats) confirmBtn.disabled = false;
       });
   }
   ```

6. **Confirm button guard** â€” In the confirm click handler, after the existing validation checks, add a check for superpower requirement:
   ```javascript
   // Validate: superpower must be selected if the book requires it
   const superpowerStep = overlay.querySelector('#cc-superpower-step');
   if (superpowerStep && superpowerStep.style.display !== 'none' && !selectedSuperpower) {
       errorEl.textContent = 'Choose a superpower to continue';
       return;
   }
   ```

7. **Pass superpower to onComplete** â€” Change the `onComplete` call from:
   ```javascript
   await onComplete(selectedBook, rolledStats, name);
   ```
   to:
   ```javascript
   await onComplete(selectedBook, rolledStats, name, selectedSuperpower);
   ```
   The 4th parameter is `null` when no superpower config exists.

   ```javascript
   if (!needsSuperpower || selectedSuperpower) {
       confirmBtn.disabled = false;
   }
   ```
   Replace the existing unconditional `confirmBtn.disabled = false;` with this check.
