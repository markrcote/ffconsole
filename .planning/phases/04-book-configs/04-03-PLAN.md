5. **Add renderBookMechanicsSection()** — New async function:
   ```javascript
   async function renderBookMechanicsSection() {
       const container = document.getElementById('book-mechanics-section');
       if (!container) return;

       if (!currentBook) {
           container.hidden = true;
           return;
       }

       const bookConfig = await getBookConfig(currentBook);

       // Check if config has any mechanics content
       const hasContent = (bookConfig.extraStats && bookConfig.extraStats.length > 0)
           || (bookConfig.resources && bookConfig.resources.length > 0)
           || (bookConfig.checklists && bookConfig.checklists.length > 0)
           || (bookConfig.superpower && state.mechanics && state.mechanics.superpower);

       if (!hasContent) {
           container.hidden = true;
           return;
       }

       const mechanicsState = state.mechanics || {};

       renderBookMechanics(container, bookConfig, mechanicsState, (updatedMechanics) => {
           state.mechanics = updatedMechanics;
           games[currentBook] = state;
           save({ games, currentBook });
       });
   }
   ```
   The `onMechanicsChange` callback updates the local state.mechanics dict and saves to backend on every change.

6. **Verify render() callers** — Check that all places calling `render()` handle the now-async return. The existing callers:
   - `init()` — already in async context, add `await render();`
   - `selectBook()` — already async, add `await render();`
   - `_applyNewCharacter()` — already async, add `await render();`
   - `syncStateFromServer()` — this one calls render() synchronously. Change to: `render();` — since render() returns a promise but the caller does not need to wait (fire-and-forget is fine here since save() is called separately).

7. **Verify load restores mechanics** — In `init()`, the existing code does:
