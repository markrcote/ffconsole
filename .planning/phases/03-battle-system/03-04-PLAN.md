Replace the stub in `js/ui/battle.js` with a full implementation. The module exports two functions: `renderBattle` and `loadCombatHistory`.

**Module structure:**

```javascript
/**
 * Battle system panel UI.
 * Self-contained module — receives container, state accessor, and callbacks.
 * Does NOT import app.js (circular import pitfall).
 */

export function renderBattle(container, getState, callbacks) { ... }
export async function loadCombatHistory(bookNumber, historyContainer) { ... }
```

**Signature:** `renderBattle(container, getState, callbacks)` where:
- `container` is the `.combat-section` element
- `getState()` returns `{ state, combatState, currentBook }` — state has `.skill`, `.stamina`, `.luck` each with `.initial`/`.current`; combatState has `.active`, `.round`, `.enemy`
- `callbacks` is `{ onStart, onRollRound, onFlee, onEnd, onStatSync, onCombatEnd }`:
  - `onStart(bookNumber, name, skill, stamina)` — calls `startCombat()` from mechanics.js
  - `onRollRound(bookNumber, playerSkill, round, enemyState)` — calls `rollCombatRound()`
  - `onFlee(bookNumber, winner, finalPS, finalES, rounds, enemyName)` — calls `endCombat()` with winner='fled'
  - `onEnd(bookNumber, winner, finalPS, finalES, rounds, enemyName)` — calls `endCombat()` with winner='player'/'enemy'
  - `onStatSync(session)` — calls `syncStateFromServer()`
  - `onCombatEnd()` — callback after combat is fully over (for app.js to reset combatState)

**What renderBattle does — bind events to existing DOM elements:**

1. **Start Combat button** (`#start-combat`):
   - Read `#enemy-name` value (default 'Enemy'), `#enemy-skill` (default 8), `#enemy-stamina` (default 8)
   - Call `callbacks.onStart(bookNumber, name, skill, stamina)` (returns session, mostly ignored)
   - Update internal state: set `combatActive = true`, `round = 1`, store enemy object `{ name, skill, stamina, staminaInitial: stamina }`
   - Show `#combat-active`, hide `#combat-setup`
   - Initialize Stamina bars: call `updateStaminaBars()` with player stamina from `getState().state.stamina` and enemy from internal state
   - Set `#combat-status` text to `Round 1`
   - Set `#enemy-stamina-label` text to enemy name
   - Clear `#combat-round-result`, `#combat-result`, `#combat-summary`
   - Unhide `#combat-summary`'s hidden attribute = true (reset from previous fights)

2. **Roll Round button** (`#roll-round`):
   - Guard: if not `combatActive`, return
   - Disable roll-round and flee buttons during the async call
   - `const { state, combatState, currentBook } = getState()`
   - Call `const r = await callbacks.onRollRound(currentBook, state.skill.current, round, enemy)`
   - Update internal enemy stamina: `enemy.stamina = r.enemyStaminaAfter`
   - Call `callbacks.onStatSync(r.session)` if `r.session` exists (patches player stamina server-side)
   - Get authoritative player stamina: `const playerStamina = r.session ? r.session.stamina.current : state.stamina.current`
   - Call `updateStaminaBars(playerStamina, getState().state.stamina.initial, enemy.stamina, enemy.staminaInitial)`
   - Render round result card into `#combat-round-result` using `renderRoundCard(round, r, enemy, playerStamina)`
   - Set `#combat-result` text (for screen readers / aria-live): brief result text
   - Check for combat end: if `playerStamina <= 0` or `enemy.stamina <= 0`:
     - Determine winner: `enemy.stamina <= 0 ? 'player' : 'enemy'`
     - Call `await callbacks.onEnd(currentBook, winner, playerStamina, enemy.stamina, round, enemy.name)`
     - Set `combatActive = false`
     - Render summary into `#combat-summary` via `renderSummary(winner, round, ...)`
     - Show `#combat-summary` (remove hidden)
     - Hide roll/flee buttons (disable them)
     - Add a "New Battle" button that resets to setup view
     - Call `loadCombatHistory(currentBook, document.getElementById('combat-history'))` to refresh history
     - Call `callbacks.onCombatEnd()`
     - Return (don't increment round)
   - Increment `round++`
   - Update `#combat-status` to `Round ${round}`
   - Re-enable buttons

3. **Flee button** (`#flee-combat`):
   - Guard: if not `combatActive`, return
   - Disable buttons
   - `const { state, currentBook } = getState()`
   - Call `const r = await callbacks.onFlee(currentBook, 'fled', state.stamina.current, enemy.stamina, round, enemy.name)`
   - Call `callbacks.onStatSync(r.session)` if `r.session` exists (applies the 2 stamina deduction from server)
   - Set `combatActive = false`
   - Get updated stamina: `const playerStamina = r.session ? r.session.stamina.current : Math.max(0, state.stamina.current - 2)`
   - Update stamina bars one final time
   - Render summary via `renderSummary('fled', round, ...)`
   - Show `#combat-summary`, add "New Battle" button, hide roll/flee
   - Call `loadCombatHistory(currentBook, document.getElementById('combat-history'))`
   - Call `callbacks.onCombatEnd()`

**Internal helper functions (not exported):**

`updateStaminaBars(playerCurrent, playerMax, enemyCurrent, enemyMax)`:
- Calculate percentages: `Math.max(0, Math.round((current / max) * 100))`

- Same for enemy bar elements

- Player skill for display: get from `getState().state.skill.current`
- Outcome text: `'player_hit'` -> "You hit! {enemy.name} takes 2 damage", `'enemy_hit'` -> "{enemy.name} hits! You take 2 damage", `'tie'` -> "Tied -- no damage"

This task implements the `loadCombatHistory` function within the same `js/ui/battle.js` file (add it after `renderBattle`).

**Signature:** `export async function loadCombatHistory(bookNumber, historyContainer)`
- `bookNumber` — current book session number
- `historyContainer` — the `#combat-history` DOM element

**Implementation:**

1. **Fetch logs:**
   ```javascript
   const res = await fetch(`/api/sessions/${bookNumber}/logs`);
   if (!res.ok) { historyContainer.innerHTML = ''; return; }
   const logs = await res.json();
   ```

2. **Filter to combat logs only:**
   ```javascript
   const combatLogs = logs.filter(l =>
       l.action_type === 'combat_start' || l.action_type === 'combat_round' || l.action_type === 'combat_end'
   );
   ```

3. **Reverse to chronological order** (GET returns newest first by id DESC):
   ```javascript
   combatLogs.reverse();
   ```

4. **Group into battles** — scan forward, each `combat_start` begins a new battle group:
   ```javascript
   const battles = [];
   let current = null;
   for (const log of combatLogs) {
       if (log.action_type === 'combat_start') {
           current = { start: log, rounds: [], end: null };
           battles.push(current);
       } else if (current && log.action_type === 'combat_round') {
           current.rounds.push(log);
       } else if (current && log.action_type === 'combat_end') {
           current.end = log;
           current = null; // battle complete
       }
   }
   ```

5. **Render battles** (most recent first — reverse the battles array for display):
   - If no battles, set `historyContainer.innerHTML = '<p class="combat-log__entry">No battles yet.</p>'` and return
   - Build HTML:
   ```javascript
   battles.reverse(); // most recent first for display
   historyContainer.innerHTML = `
       <div class="combat-log">
           <div class="combat-log__title" id="combat-log-toggle">Past Battles (${battles.length})</div>
           <div class="combat-log__battles" id="combat-log-list" hidden>
               ${battles.map(b => renderBattleEntry(b)).join('')}
           </div>
       </div>
   `;
   ```
   - Bind toggle: `#combat-log-toggle` click toggles `hidden` on `#combat-log-list` and toggles `expanded` class on the title

6. **`renderBattleEntry(battle)` helper** (internal function):
   - Extract from `battle.start.details`: `enemy_name`, `enemy_skill`, `enemy_stamina`
   - Extract from `battle.end.details` (if present): `winner`, `rounds` count
   - Determine outcome text: winner='player' -> "Victory", winner='enemy' -> "Defeat", winner='fled' -> "Fled"
   - If no `battle.end`, show "In progress" (for battles interrupted by page close)
   - Render:
   ```html
   <div class="combat-log__battle">
       <div class="combat-log__battle-header">
           vs {enemy_name} (Skill {enemy_skill}, Stamina {enemy_stamina}) — {outcome}
       </div>
       {battle.rounds.map(r => renderRoundEntry(r)).join('')}
   </div>
   ```

7. **`renderRoundEntry(roundLog)` helper:**
   - Extract from `roundLog.details`: `round`, `player_attack`, `enemy_attack`, `result`, `enemy_stamina_after`
   - Result text: `player_hit` -> "You hit", `enemy_hit` -> "Enemy hit", `tie` -> "Tie"
   - Render: `<div class="combat-log__entry">R{round}: AS {player_attack} vs {enemy_attack} — {result text}</div>`

**Error handling:**
- Wrap the entire function in try/catch. On error, set `historyContainer.innerHTML = ''` (fail silently — history is non-critical).
- If `bookNumber` is falsy, return immediately without fetching.
