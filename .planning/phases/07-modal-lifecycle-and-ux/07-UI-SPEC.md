## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, gap between buttons |
| md | 16px | Default element spacing, section internal padding |
| lg | 24px | Section padding, modal internal padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |

Exceptions:
- Touch targets: minimum 44px height for all interactive elements (mobile-first requirement). Existing `.mechanic-btn` uses `min-height: 48px` — maintain this floor.

- Overlay padding: 20px (existing `.modal-overlay` rule) — retain.
- Shake animation translation offsets: ±8px, ±6px, ±4px — sub-grid values used only within keyframe; not a spacing token.

Source: `css/style.css` — existing measured values; mobile-first constraint from `CLAUDE.md`.

## Animation Contract

All new animations are CSS-only (`@keyframes`). No JS animation libraries.

### Modal Open — Slide Up

- Keyframe name: `modal-slide-up`
- Applied to: `.modal` element inside `.modal-overlay.active` on the frame the overlay becomes active
- Transform: `translateY(100%)` → `translateY(0)`
- Opacity: 1 throughout (no fade on open — slide only)
- Duration: 300ms
- Easing: `ease-out`
- Trigger: Add class `.modal-overlay--opening` immediately on open; remove after `animationend`

### Modal Close — Fade Out

- Keyframe name: `modal-fade-out`
- Applied to: `.modal-overlay` when closing (add class `.modal-overlay--closing`)
- Opacity: 1 → 0
- Duration: 300ms
- Easing: `ease-in`
- Trigger: Add `.modal-overlay--closing` in `closeBattleModal()`; call `overlay.remove()` in `animationend` listener

### Dismiss Guard — Shake

- Keyframe name: `modal-shake`
- Applied to: `.modal` (inner box, not the overlay) when backdrop or Escape pressed during active combat
- Transform sequence: `translateX(0)` → `translateX(-8px)` → `translateX(8px)` → `translateX(-6px)` → `translateX(6px)` → `translateX(-4px)` → `translateX(0)`
- Duration: 300ms total
- Easing: `ease-in-out`
- Trigger: Add class `.modal--shake`; remove class in `animationend` listener (ensures re-triggerable)
- No text, no sound, no toast — purely physical feedback (per D-02)

### Reduced Motion Override

```css
@media (prefers-reduced-motion: reduce) {
    /* Skip slide-up: modal appears instantly */
    .modal-overlay--opening .modal { animation: none; }
    /* Skip fade-out: overlay disappears instantly */
    .modal-overlay--closing { animation: none; opacity: 0; }
    /* Skip shake: no movement feedback */
    .modal--shake { animation: none; }
}
```

Source: D-04, D-05 from `07-CONTEXT.md`.

## Scroll Lock Contract

Technique: `position:fixed` + saved `scrollY` (iOS Safari safe, per D-03).

On open:
1. `savedScroll = window.scrollY`
2. `document.body.style.cssText = 'position:fixed; top:-${savedScroll}px; width:100%;'`

On close:
1. `document.body.style.cssText = ''`
2. `window.scrollTo(0, savedScroll)`

`savedScroll` is a module-level variable in `battleModal.js` (not a closure — must survive between `open` and `close` calls).

Source: D-03 from `07-CONTEXT.md`.

## Focus Management Contract

On open:
- Target element: `#enemy-name` (the enemy name text input, first interactive element)
- Timing: inside `requestAnimationFrame()` callback after `document.body.appendChild(overlay)` to ensure element is in DOM
- Implementation: `overlay.querySelector('#enemy-name').focus()`

On close:
- Return focus to the "Start Battle" trigger button in the adventure sheet
- Target: `document.getElementById('start-battle-btn')` (the trigger button in `index.html`)
- Timing: after `overlay.remove()`

Full Tab trap and `aria-modal` are explicitly deferred (documented in `REQUIREMENTS.md` Future Requirements). Phase 7 does not implement tab trapping.

Source: D-06 from `07-CONTEXT.md`; Future Requirements from `REQUIREMENTS.md`.

## Dismiss Guard Contract

| State | Escape key | Backdrop tap |
|-------|------------|--------------|
| Setup form visible (combat not started) | Close modal (free dismiss) | Close modal (free dismiss) |
| Combat active (`combatActive === true`) | Play shake animation, no close | Play shake animation, no close |

`combatActive` signal: `battleModal.js` maintains a local `let combatActive = false` boolean, flipped by the `onCombatStateChange(active)` callback added to the `callbacks` object passed to `renderBattle()` (per D-01).

Escape handler: added as `keydown` listener on `document` when `openBattleModal()` runs; removed in `closeBattleModal()`. Mirrors the existing pattern at `app.js:440–445`.

Backdrop handler: `click` on the `.modal-overlay` element (not the `.modal` inner box); checked via `event.target === overlay`.

Source: D-01, D-02 from `07-CONTEXT.md`.
