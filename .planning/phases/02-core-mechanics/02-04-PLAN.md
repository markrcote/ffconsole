# Plan 4: `js/ui/diceRoller.js` — Standalone Dice Roller Widget

**Phase:** 02 — Core Mechanics
**Goal:** Implement the standalone dice roller section: "Roll d6" and "Roll 2d6" buttons with individual die face display and total.
**Requirements:** DICE-01, DICE-02
**Depends on:** Plan 2 (CSS classes must exist)

## Context

`js/ui/diceRoller.js` is a stub that exports an empty `renderDiceRoller(container)`. This plan replaces the stub with a complete implementation. The widget is fully self-contained — it receives the section container element, builds its HTML content, and binds click events. No state, no callbacks to `app.js`. It imports only `roll` from `js/dice.js`. Per D-10, this is a separate section from Tests. Per D-11, 2d6 shows two `.die-face` elements plus the total; 1d6 shows one `.die-face`.

## Tasks

### Task 1: Implement `renderDiceRoller`

**File:** `js/ui/diceRoller.js`
**Action:** Modify (replace stub body entirely)

Replace the entire file with:

```javascript
/**
 * Standalone dice roller widget.
 * Phase 2 implementation.
 *
 * renderDiceRoller(container) — renders into the provided section element.
 * Called once from app.js init() with document.getElementById('dice-section').
 * No state, no callbacks. Imports only roll() from dice.js.
 */

import { roll } from './dice.js';

/**
 * Render the dice roller widget into the given container element.
 * The container is the <section class="mechanics-section dice-section"> element.
 * The section's <h2> title is already in index.html; this function fills the body.
 * @param {HTMLElement} container
 */
export function renderDiceRoller(container) {
    if (!container) return;

    // Build inner HTML: two buttons + result area
    container.insertAdjacentHTML('beforeend', `
        <div class="dice-roller-buttons">
            <button class="mechanic-btn" id="roll-d6">Roll d6</button>
            <button class="mechanic-btn" id="roll-2d6">Roll 2d6</button>
        </div>
        <div class="dice-roller-result" id="dice-roller-result" aria-live="polite"></div>
    `);

    const resultEl = container.querySelector('#dice-roller-result');

    // ── Roll d6 ──────────────────────────────────────────────────────────────
    container.querySelector('#roll-d6').addEventListener('click', () => {
        const value = roll(6);
        // Single die face (DICE-02)
        resultEl.innerHTML = `<span class="die-face">${value}</span>`;
    });

    // ── Roll 2d6 ─────────────────────────────────────────────────────────────
    container.querySelector('#roll-2d6').addEventListener('click', () => {
        const d1 = roll(6);
        const d2 = roll(6);
        const total = d1 + d2;
        // Two die face elements plus total (DICE-02, D-11)
        // Format: [d1] [d2] = total
        resultEl.innerHTML = `
            <span class="die-face">${d1}</span>
            <span class="die-face">${d2}</span>
            <span class="dice-total">= ${total}</span>
        `;
    });
}
```

No additional helpers are needed. The `roll` import from `js/dice.js` is the only dependency.

**Verification:** The file exports `renderDiceRoller` as a named function. After Plan 5 wires it up:
- Clicking "Roll d6" renders one `.die-face` bubble with a value 1–6 in `#dice-roller-result`.
- Clicking "Roll 2d6" renders two `.die-face` bubbles and a `.dice-total` span showing the sum in `#dice-roller-result`.

Pre-wiring manual check: `import('./js/ui/diceRoller.js').then(m => console.log(typeof m.renderDiceRoller))` in browser DevTools returns `"function"`.
