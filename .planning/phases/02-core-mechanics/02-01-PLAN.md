# Plan 1: Backend — Expose `name` in Session API

**Phase:** 02 — Core Mechanics
**Goal:** Add `name` to the session schema and write it through the full PUT/PATCH pipeline so character names can be persisted.
**Requirements:** CHAR-03
**Depends on:** None

## Context

`Session.name` column already exists in the ORM model (`backend/models.py` line 26: `name = Column(Text, nullable=True)`). However, none of the Pydantic schemas include it, and neither the PUT upsert nor the PATCH handler writes it. This plan closes the gap so the frontend can send and receive a `name` field through the standard `/api/sessions/{book}` PUT and PATCH endpoints. Without this, Phase 2 frontend code cannot persist character names.

## Tasks

### Task 1: Add `name` to Pydantic schemas

**File:** `backend/schemas.py`
**Action:** Modify

Add `name: str | None = None` to three schema classes:

1. `SessionCreate` (line 41) — add after `book_number`:
   ```python
   name: str | None = None
   ```

2. `SessionResponse` (line 11) — add after the `mechanics` field:
   ```python
   name: str | None = None
   ```

3. `SessionUpdate` (line 49) — add after `mechanics`:
   ```python
   name: str | None = None
   ```

4. In `SessionResponse.assemble_stat_blocks` validator (lines 23–38), add `"name": data.name,` to the returned dict after `"mechanics"`:
   ```python
   "mechanics": json.loads(data.mechanics_json or '{}'),
   "name": data.name,
   ```

This is the only change to `schemas.py`. All three classes already follow the same pattern; `name` is appended as the last field.

**Verification:** `python -c "from backend.schemas import SessionCreate, SessionResponse, SessionUpdate; print('ok')"` runs without error from the project root.

### Task 2: Write `name` in upsert and patch handlers

**File:** `backend/routers/sessions.py`
**Action:** Modify

1. **`upsert_session` handler** (lines 55–80): after `session.mechanics_json = ...` in both the `if session:` branch (update) and the `else:` branch (create), add `session.name = body.name`.

   In the update branch (after line 64):
   ```python
   session.name = body.name
   ```

   In the create branch (in the `Session(...)` constructor call, add):
   ```python
   name=body.name,
   ```

2. **`patch_session` handler** (lines 83–102): after the `if body.mechanics is not None:` block (before `session.updated_at = _now()`), add:
   ```python
   if body.name is not None:
       session.name = body.name
   ```

No other changes to `sessions.py`.

**Verification:** Start the server (`uvicorn backend.main:app --host 0.0.0.0 --port 3000`) and run:
```
curl -s -X PUT http://localhost:3000/api/sessions/1 \
  -H 'Content-Type: application/json' \
  -d '{"book_number":1,"name":"Alaric","skill":{"initial":10,"current":10},"stamina":{"initial":20,"current":20},"luck":{"initial":9,"current":9},"mechanics":{}}' \
  | python3 -m json.tool
```
Response JSON must include `"name": "Alaric"`. Then verify PATCH:
```
curl -s -X PATCH http://localhost:3000/api/sessions/1 \
  -H 'Content-Type: application/json' \
  -d '{"name":"Raven"}' \
  | python3 -m json.tool
```
Response JSON must include `"name": "Raven"`.

## Success Criteria

- `SessionCreate`, `SessionResponse`, and `SessionUpdate` all include `name: str | None = None`
- `PUT /api/sessions/{book}` writes `name` to the database and returns it in the response
- `PATCH /api/sessions/{book}` updates `name` when provided and returns it in the response
- `GET /api/sessions/{book}` returns `name` in the response body (via the updated `assemble_stat_blocks` validator)
- A session created or updated with `name: null` stores `null` without error
